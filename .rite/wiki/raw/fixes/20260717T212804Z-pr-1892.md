### Fix Patterns
- セキュリティゲートの verb 撤去では「subcommand 照合」だけでなく「invocation 正規化・global-flag 正規化・dequote」の 3 点を撤去前後で漏れなく復元しないと、部分的にしか塞がらない。cycle 1 の (N) は照合だけ戻し正規化を落としたため path/backslash invocation と quoted sub-action が素通りした。
- git 機械ゲートの invocation 検出は basename (`git|*/git`) で判定し bare `git` に正規化すべき。`/usr/bin/git` / `./git` は bare `git` トークンを持たない。
- separate-arg global flag (`-C` / `--git-dir` / `--work-tree` / `--namespace` / `--attr-source` / `--super-prefix`) は skip_arg で値を捨てないと、値が subcommand slot を吸収して穴になる。arg-taking flag 集合は一度で完全に列挙する (whack-a-mole を避ける)。
- token 再構築は subcommand も後続トークンも dequote 済みで積む (非対称にすると quote 装飾で bypass される)。
- 撤去 refactor の再配分検証は dev 版/PR 版フックへ crafted stdin を与える revert test (deny/allow 比較) が最も確実。
