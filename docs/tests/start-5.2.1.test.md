# テスト仕様書: start.md Phase 5.2.1 チェックリスト完了確認

## 概要

`plugins/rite/commands/issue/start.md` の Phase 5.2.1「チェックリスト完了確認（PR 作成前バリデーション）」に対するテスト仕様書。
Issue #398 で導入された、Issue 本文のチェックリストが全て完了するまで PR 作成に進まないガード機能の検証。

## 対象機能

### Phase 5.2.1 チェックリスト完了確認

PR 作成前に Issue 本文のチェックリストの完了状況を確認し、未完了項目がある場合は実装作業の継続を促す。

**処理フロー:**

```
Phase 5.2 (lint) 完了
    ↓
Phase 5.2.1 チェックリスト確認
    ↓
チェックリストあり？
├─ No → Phase 5.3 へスキップ
└─ Yes → 全項目完了？
    ├─ Yes → Phase 5.3 へ進む
    └─ No → 未完了項目を表示し、Phase 5.1 へ戻る
              （実装作業を継続）
```

---

## テストケース

### TC-001: チェックリストなしの場合のスキップ確認

**目的**: Issue 本文にチェックリストがない場合、Phase 5.2.1 がスキップされて Phase 5.3 に進むことを確認する。

**前提条件**:
- Issue 本文にチェックリスト（`- [ ]` 形式）が含まれていない
- Phase 3.6（Issue 情報の取得・解析）でチェックリストが抽出されていない

**検証項目**:

| # | 検証項目 | 期待値 | 結果 |
|---|---------|--------|------|
| 1 | Phase 5.2.1 の実行 | スキップされる | - |
| 2 | 次のフェーズ | Phase 5.3（PR 作成）に進む | - |
| 3 | ユーザーへの確認 | なし（自動で進む） | - |

### TC-003: 未完了項目がある場合のブロック確認

**目的**: 未完了のチェック項目がある場合、PR 作成に進まず実装継続を促すことを確認する。

**前提条件**:
- Issue 本文にチェックリストがある
- 一部の項目が `[ ]`（未完了）のまま

**入力例（Issue 本文）**:

```markdown
## チェックリスト

- [x] 実装完了
- [ ] テスト追加/更新
- [ ] ドキュメント更新
```

**期待される出力**:

```
⚠️ Issue チェックリストに未完了の項目があります。

| # | タスク | 状態 |
|---|--------|------|
| 1 | 実装完了 | ✅ |
| 2 | テスト追加/更新 | ⬜ |
| 3 | ドキュメント更新 | ⬜ |

未完了: 2 件

全てのチェック項目を完了してから PR を作成してください。
対応が必要なタスク:
- テスト追加/更新
- ドキュメント更新

実装作業を継続してください。完了後、再度 lint を実行すると PR 作成に進みます。
```

**検証項目**:

| # | 検証項目 | 期待値 | 結果 |
|---|---------|--------|------|
| 1 | 未完了項目の検出 | 2 件検出 | - |
| 2 | 未完了項目の表示 | テーブル形式で表示 | - |
| 3 | 次のフェーズ | Phase 5.1 に戻る（PR 作成には進まない） | - |
| 4 | メッセージ内容 | 「実装作業を継続してください」 | - |

## 手動検証手順

### 検証 1: 未完了項目がある場合の動作

1. チェックリスト付きの Issue を作成:
   ```
   ## チェックリスト
   - [ ] 実装完了
   - [ ] テスト追加
   ```

2. `/rite:issue:start {issue_number}` を実行

3. 実装を完了し、「実装完了」のみチェック

4. lint 通過後、Phase 5.2.1 で以下を確認:
   - 「テスト追加」が未完了として表示される
   - PR 作成に進まず、実装継続のメッセージが表示される

### 検証 2: 全項目完了後の動作

1. 残りのタスク（テスト追加）を実装

2. コミット・プッシュ

3. lint を再実行

4. Phase 5.2.1 で以下を確認:
   - 全項目完了と判定される
   - Phase 5.3（PR 作成）に進む
