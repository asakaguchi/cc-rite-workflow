### Finding Patterns
コマンド定義 .md 内の埋め込み bash スクリプトで頻出する 3 パターンを検出:
1. gh 呼び出しの `2>/dev/null` 全捨てによる silent failure — reason が固定文字列に潰れ真因を診断不能。sibling スクリプト (watchdog-status-mismatch.sh / link-sub-issue.sh / projects-status-update.sh) は stderr を tempfile 退避する確立慣習を持つが、新規スクリプトがそこから逸脱しやすい。
2. `gh api graphql` の `.errors[]` 非検査 — HTTP 200 + partial errors を握り潰し、欠損データを権威データとして提示。link-sub-issue.sh は errors を明示検査する慣習。
3. CRITICAL「verbatim 実行・alter no line」警告と `{placeholder}` 置換必須の矛盾 — close.md / cleanup.md は verbatim ブロック直前に placeholder 置換 carve-out を明記する確立パターンを持つ。
4. `gh api graphql` の field flag は `-F` (typed coercion) ではなく `-f` (raw string) を opaque な ID!/String! 変数に使うのが Phase 5 系の規約。
