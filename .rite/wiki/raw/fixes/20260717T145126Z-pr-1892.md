### Fix Patterns
- セキュリティゲートの verb 撤去 refactor では「撤去集合に維持すべきカテゴリが混在していないか」を撤去前後の deny/allow ハーネス比較で検証すべき。本 PR では (A) 一括撤去が working-tree verb (撤去対象) と native .git-write subcommand (維持対象 RCE 経路) を巻き添えにしていた。
- .git 書き込みゲートの subcommand 検出は global-flag 正規化が必須。`git -C x config` / `git -c key=val <cmd>` は subcommand が git 直後に来ないため、naive な "git 直後の subcommand" 照合を bypass する。撤去された (A)–(G) の global-flag 正規化がこれを担っていた。
- inline `git -c core.hooksPath=… <cmd>` は subcommand 無しで hook を仕込む RCE ベクタ。config write の一律 deny (read 形は allow-list) に含めるべき。
