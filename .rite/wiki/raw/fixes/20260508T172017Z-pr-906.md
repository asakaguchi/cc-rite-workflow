### Fix Patterns

3 件 LOW finding すべてを単一 commit で修正 (ratchet test の堅牢性向上系):

1. **下限 assert の単位統一**: `grep -c` (line 単位) と `grep -oE | wc -l` (occurrence 単位) の混在
   を後者に統一。1 行に複数出現する phrase の集約 slim を後続 PR で行う際、ratchet 漏れを構造的に防ぐ。

2. **bash block 終端の動的追跡**: 固定 +7 行 window を awk による ``` 終端追跡に変更。
   line continuation 長さ依存の脆弱性を排除し、将来 `--next` 等の長文化に追随可能に。

3. **書式前提のコメント明示**: 引数 regex `${flag}([[:space:]]|$)` が前提とする `--flag value`
   形式 (vs `--flag=value` 形式) を test 内コメントで明示。書式拡張 PR が出た際の TODO とする。

### Effective Patterns

- **動的 block 抽出**: awk の state machine で in_block / in_create flag を組み合わせ、bash block 終端
  まで読む方式は `\0` 区切り出力 + `read -d ''` で受ける portable な技法 (固定行数 window より堅牢)。
- **occurrence vs line 単位の選択**: ratchet test では occurrence 単位 (`grep -oE | wc -l`) が
  原則。line 単位 (`grep -c`) は読みやすさ優先のスニペットに留める。
