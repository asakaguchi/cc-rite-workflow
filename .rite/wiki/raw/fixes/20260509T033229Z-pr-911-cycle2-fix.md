### Fix Patterns
cycle 2 fix で `^[[:space:]]*[^[:space:]#].*literal_X` 形式を前置 not-match `!/^[[:space:]]*#/ && /literal_X/` 形式に refactor。前者は X が行頭から始まる行を `[^[:space:]#]` が先頭の文字を消費した結果、続く literal X が再発見できず silent miss する backtracking 経路を持つ。確立された bash/awk regex のイディオム (前置 not-match) は意図が直接的で X の行頭出現有無に影響を受けない。

教訓: 「shell コメント除外」のような単純な意図でも、negative-class + greedy `.*` + literal pattern の組み合わせは backtracking trap を作り得る。前置 not-match `!/^pat/` は除外意図を直接表現し silent miss を構造的に防ぐ。
