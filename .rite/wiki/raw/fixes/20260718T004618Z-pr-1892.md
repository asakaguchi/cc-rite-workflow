### Fix Patterns
- コマンド文字列を平坦化して substring マッチする allow-list ゲートは、複合コマンドで allow token が別の write occurrence を masking する構造的欠陥を持つ。per-occurrence の state machine で各 invocation を独立評価し、read/write を「直後トークン」で判定すべき。
- allow-list ゲートの default 方向が重要: allow-by-default (read substring があれば行全体を exempt) は masking leak を生む。deny-by-default (read allow-list 該当のみ通過、それ以外 deny) に反転すると想定外 form が leak せず over-block に倒れ収束する。
- 同種の判定 (config の read/write, remote の mutating/read) は fail-closed default を対称に適用すべき。片方 fail-closed・片方 fail-open だと enumeration fragility の非対称が残り reviewer に指摘される。
- 機械ゲートの「収束済み」判断は慎重に。新しい構造クラスの bypass が出たら前提が崩れた旨をユーザーに訂正・再確認する (silent fix-and-continue しない)。
