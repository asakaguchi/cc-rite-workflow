### Fix Patterns
bash cleanup/GC スクリプトで find を process substitution `< <(find ...)` で呼ぶと subshell の exit code がシェルに伝播せず、find の wholesale 失敗 ($TMPDIR 不在/権限なし/IO エラー) が空ループ→無言 no-op になる。同一スクリプト内の sibling ブロックが `if cmd=$(...); then ... else rc=$?; WARNING; errors++; fi` で rc 捕捉している場合、新規ブロックも command substitution + here-string `<<<` 形式に揃えるのが対称性を保つ正しい修正。空 stdout 時の here-string 単一空行はループ先頭の `[ -z ]` ガードで skip する (branch-loop と同じイディオム)。failure path のテストは TMPDIR を不在パスに向けて wholesale 失敗を誘発することで可能 (script の mktemp が /tmp 直書きなら TMPDIR override の影響を受けない)。
