### Lessons (anti-patterns reinforced)

- **Anchor link integrity verification protocol** (新規): heading に slug を含む anchor を手書きで参照する PR では、commit 前に必ず GitHub の slug 算法 (lowercase / 非英数除去 / space→hyphen) で再計算し、本文中の anchor と一致するか verify する。typo は silent failure で人間レビュアーが click するまで発覚しない。

- **Multi-symbol decomposition の sentinel grep verification** (Terminal marker 系で再確認): 1 つの orchestrator を N 個に分解する PR では、ドキュメント側の Terminal marker / sentinel 列に **新コマンドの実装ファイル内 sentinel literal** を grep verify する。旧 orchestrator の用語を流用すると AC-4「未実装機能の記述が無いこと」違反になりやすい。

- **CHANGELOG-driven cross-key DEPRECATED 化 protocol** (Asymmetric Fix Transcription extension): CHANGELOG に `removed in #N` を追記する PR では、その # で削除された全 keys を CHANGELOG 自体から再抽出し、各 key について `grep -rn '<key>' docs/` で残存する live citation を機械検出し、同じ DEPRECATED 化パターンで一括更新する。1 つだけ更新して他を漏らすパターンは review で必ず発覚するため、最初から網羅的に処理すべき。
