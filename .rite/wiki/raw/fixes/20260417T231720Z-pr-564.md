### Fix Patterns

- **Phase asymmetry 解消**: 新規追加 bash block に既存 Phase の `case "$branch_strategy"` 2 分岐構造 (separate_branch=git show / same_branch=cat) を後付けで移植する pattern。legitimate absence の stderr pattern matching も両経路に拡張 (ENOENT / No such file)。Phase 2.2/2.3/6.0/8.2 は全て 2 分岐を持つため、新 Phase 追加時は「既存 Phase の分岐構造と対称か」を cross-file check リストに含める。
- **pipefail per-stage stderr 共有**: `cmd1 2>"$err" | cmd2 2>>"$err"` 形式で両 stage の stderr を同一 tempfile に append する pattern。片側のみ `2>` で退避すると pipefail 下で反対側 stage 失敗時に `head -3 "$err"` が silent 空になる diagnostic gap が発生する (Phase 6.0 awk_sort_err と Phase 6.2 sort_err の非対称事例)。
- **行番号ハードコード drift の根本解消**: intra-file / cross-file の `L{N}` 形式参照は新規機能追加で確実に drift する。Phase 番号 + semantic 名 (section 見出し / enum 変数名 / 機能説明) での参照に一元化する。旧実装で 7 件の stale 行番号参照 (Phase 8.2 L1011 が実 L1041、L759 が blank line、L508-511 が comment block 等) を一括駆逐した。
- **dead path 検出**: prose 契約 (「`実行失敗` を含む WARNING」) と実 emit 文字列の grep による整合性検証。lint.md が emit する `に失敗しました` / `ERROR:` に揃えることで step 1 のデッドパスを解消。prose を書く際は emit 側の実 grep を必須とする運用。
- **function 名 scope prefix**: `_rite_<scope>_<phase>_cleanup` 2 階層命名に統一 (scope 不在だと `_rite_phase2_cleanup` が複数 site で衝突するため)。`<phase>` は小数点を除いた連結形式 (Phase 6.2 → `phase62`)。
