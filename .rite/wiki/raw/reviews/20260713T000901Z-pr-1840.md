### Finding Patterns
- **「安全側判定」の実装は git の縮退挙動で反転する**: `git diff --quiet <rev> -- <pathspec>` は pathspec が実ファイルに不一致だと「差分なし = exit 0」を返す。「不一致なら安全側 (divergent) に落ちる」という直感は真逆で、quotePath の C-quote / cwd 相対解決 / 空 pathspec のいずれでも「比較していないのに同一」と誤判定し、破棄承認でデータ喪失に至る経路を 3 回別形で作った (F-06 cwd / F-07 空 / F-10 quote)。教訓: pathspec 経由の否定的判定 (差分なし) を安全性の根拠にするときは「pathspec が実在ファイルに解決されたこと」を独立に保証する。
