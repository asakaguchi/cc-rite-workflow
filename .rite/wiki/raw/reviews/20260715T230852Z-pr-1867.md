### Finding Patterns
- unquote 単語分割ループ (`for tok in $VAR`) の noglob 化は既存の兄弟実装 ((H) tokenizer) と同型パターンで水平展開するのが定石。save/restore は `case $- in *f*)` で直前状態を捕捉し、ループ後 `[ "$was_set" = "1" ] || set +f` で復元する。変数名はブロック接頭辞で衝突回避 (`_gd_` / `_wt_`)。
- over-DENY 回帰テストは「fail-on-revert」を実機で確認するのが必須 (テストが discriminating か: 未修正で DENY・修正後 ALLOW)。CWD にフラグ名ファイル (`-b`) を full path で作成し、glob を含む正当コマンドが over-DENY されないことを固定する (姉妹 TC-125 と同型)。
- `set -f` window は最小限に (対象ループ本体のみ)。`case`/`[[ == ]]` は noglob 非影響のため判定ロジックは不変。ループに `break`/`return`/`exit` が無いことを確認すれば restore は常時到達する。
- レビュー観点として「塞ぎ損ねた別ループ」の網羅確認が有効: ファイル全体の unquoted `for X in $...` を grep で列挙し、全て noglob 化 or iteration-cap 済みかを確認する。
