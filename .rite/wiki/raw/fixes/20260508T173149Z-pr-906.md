#### MEDIUM finding (mutation test で実証された design flaw)

**awk state machine の multi-create-per-block blind spot**: 同一 bash block 内に複数 `flow-state-update.sh create` 呼び出しがある場合、`in_create` flag が reset されず block が連結される問題。test contract「all 32 invocations have 5 args」と実装「union of args per block is complete」の乖離。

**修正**: create 検出時に既に `in_create=1` なら前 block を `printf "%s%c", block, 0` で先 flush し、新 block を `block=$0` で開始する。mutation test (1 block 内 2 creates、2 つ目で --phase 欠落) で `total=2 asymmetric=1` を確認。

#### Cycle 1 修正スコープ漏れの完了

**🚨 上限 assert の単位混在**: cycle 1 で下限 2 件 (ask_count, mandatory_count) を occurrence 単位に統一したが、上限 🚨 のみ `grep -c` (line 単位) のまま残留。reviewer 全員が独立検出 (高信頼度合意)。

**修正**: `bell_count=$(grep -oE '🚨' ... | wc -l | tr -d ' ')` に統一。実測 line=35 → occurrence=41 (差 6)。
