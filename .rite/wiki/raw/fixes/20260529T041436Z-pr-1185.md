### Fix Patterns
コマンド定義 .md 内の埋め込み bash スクリプトに対する 4 件の修正パターン:
1. gh 呼び出しの `2>/dev/null` を tempfile 退避 (`2>"$err"`) に変更し、失敗 reason に実 stderr を含める。固定文字列 reason は silent failure の温床。
2. `gh api graphql` の `.errors[]` を items guard の前に検査し、HTTP 200 + partial errors を fail-fast で fallback に倒す。`.data.node.items` の存在チェックだけでは partial response を握り潰す。
3. CRITICAL「verbatim 実行」警告と `{placeholder}` 置換必須の矛盾は、verbatim ブロック直前に「placeholder のみ置換可、query/jq 本文は変更不可」の carve-out を明記して解消 (close.md/cleanup.md の確立パターン)。
4. `gh api graphql` の opaque string 変数 (ID!/String!) には `-F` (typed coercion) ではなく `-f` (raw string) を使う。同一ファイル内の既存規約 (Phase 5 系) との整合を優先。
