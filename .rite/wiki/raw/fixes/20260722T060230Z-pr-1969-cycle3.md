### Fix Patterns

テストの前提条件検証パターン: chmod によるパーミッション制限をテストフィクスチャで使う際、chmod がその環境（root実行、WSL2 DrvFs等の一部ファイルシステム）で実際に強制されるかを probe write で事前検証してからアサーションに進むべき。「制限をかけたつもり」で直接アサーションすると、制限が効かない環境で偽陽性 pass（false positive、バグを検出できないのに green になる）が生じる。probe 自体のエラーメッセージ抑制にはブレースグループ `{ : > file; } 2>/dev/null` を使う（単純な `: > file 2>/dev/null` だとリダイレクトの open 失敗メッセージが抑制されないケースがある）。
