### Lessons (anti-patterns reinforced — cycle 2 specific)

- **CHANGELOG fact-checking with `ls` / `git show`**: When adding CHANGELOG entries for `removed in #N` or `migrated in #N` deletion-type changes, **always verify with file system check** (`ls path/`) or git inspection (`git show {sha}:path/`). This PR's cycle 1 reviewer missed both the i18n/ja "remain on disk" and project.type CONFIGURATION docs drift because the reviewer trusted the PR's CHANGELOG text without verifying against actual state. cycle 2 detected via grep + ls.

- **Asymmetric Fix Transcription (re-confirmed for v0.5.0 docs PR, cycle 2 variant)**: Same as cycle 1 F-04/F-05 pattern but at a different level. cycle 1 caught CONFIGURATION docs drift (DEPRECATED-ifying observed_likelihood_gate etc.). cycle 2 caught the LARGER drift: the CHANGELOG declared project.type retired but the docs were never updated. Lesson: when CHANGELOG adds `removed in #N` entry, the SAME PR must complete the full DEPRECATED-ification cycle for ALL keys/features mentioned. The `# wave` (e.g., #1118) is the unit of consistency, not individual keys.

- **Multi-PR drift surfaced by docs review**: When a deletion PR (#1117 / #1118) deletes implementation but doesn't update all consumer docs, the drift only surfaces during release-prep doc review (this PR). The lesson is to make deletion PRs include CONFIGURATION.md / style-guide drift fix as mandatory acceptance criteria, not deferred to release-time docs review.
