### Fix Patterns

`status_json=$(cmd) || status_json=""` という一見安全なフォールバックが、cmd の非ゼロ終了時に stdout へ既に出力済みの診断JSONを空文字列で上書き・破棄してしまうパターンを2箇所で修正した。command substitution はサブプロセスの exit code に関係なく stdout を正しくキャプチャするため、後続の `case` 文で catch-all 処理をしている限りこの種のフォールバックは不要かつ有害（診断情報の損失）である。

また、あるBashツール呼び出しでset した素のシェル変数を、別のBashツール呼び出し（別ステップ）が参照しようとする典型的な罠（`projects_status_updated`）も検出・修正した。rite workflow の SKILL.md 実行モデルでは、ステップ間の値の受け渡しは必ず `[CONTEXT] KEY=value` sentinel の echo を経由する必要がある。
