### Summary

`/rite:pr:cleanup` workflow を Simplification Charter に準拠して slim 化する Phase A-C
構成の Issue。実装作業は 7 PR (Phase A: charter 適用宣言、Phase B1-B4: references/ 4 ファイル

### 学び・知見

- **Phase 単位の分割は scope drift 防止に有効**: 1 PR 1 ファイル / 1 焦点 を厳格に守ったことで、
  scope 外指摘を follow-up Issue 化せず確実に断る運用が機能した。
- **AC 単独達成困難な目標は別 Issue 化が正解**: PR #891 マージ時点で cleanup.md は 1810 行で
  AC-3 (≤1500 行) 未達成だったが、Phase D を別 Issue (#892) として起票する判断が正しかった。
  単一 Issue で全 AC を抱え込むと commit 単位が肥大化し review が困難になる。
- **後続 Issue 完了で本 Issue の AC が事後的に満たされるパターン**: #845 自身の PR は全マージ
  済みだが、cleanup.md の slim は #892 Phase D の作業で達成された。Issue close 時には現状の
  ファイル状態で AC を再検証することで、Issue 間の依存関係を解きほぐして close できた。
- **PR description の close キーワード省略は意図的**: Phase B/C の各 PR は `Refs #845` のみ記載し
  `Closes #845` を意図的に省略していた (PR 本文に明記)。Pattern B (PR 存在も auto-close なし)
  → manual close で対応する運用が確立されている。

### 有効だったアプローチ

- 違反多い順 (B1: 19件 → B2: 9件 → B3: 4件 → B4: 2件) の PR 順序により、early review pattern
  確立 + 大きな効果を先行獲得。
- Issue body の "Scope 外指摘ハンドリングポリシー" セクションを起票時に明記したことで、
  reviewer / セルフレビュー双方が「本 Issue Scope 外につき対応見送り」と即座に判断できた。
