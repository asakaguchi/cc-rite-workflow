### Fix Patterns

- **Markdown 番号付きリスト分断の解消**: `1. ... 5.` 連番の途中に table / `**注意**:` paragraph が挟まると GFM では新規リストとして render される (CommonMark/GFM 仕様)。canonical fix は (a) item を `**xxx**: 簡潔な説明` の 1 行形式に保つ、(b) 補足情報 (table / 詳細注釈) を別 h4 sub-heading として list 後に切り出す、の 2 通り。本 PR では (b) を採用し、`#### アクション判定表 (Step 4 用)` として独立化することで list 連続性を保ちつつ補足情報を保全。
- **Canonical source 宣言の明示**: 同一 placeholder の値生成手順が複数 site に分散している場合、片方を「canonical source」と明示し、他方を「summary / cross-reference」位置付けに整理することで drift を抑制。本 PR では 4.3 を canonical、Phase 5.3 placeholder 表と #941 fix 注釈を summary に整理。
- **意図的重複の visibility 注釈**: Wiki 経験則「Asymmetric Fix Transcription」に基づく 3-site 同期契約 (本 PR の 4.3 / Phase 4 overview list / Phase 5.3 前方参照) は、両端の site で「他方への参照」blockquote を残すと drift 防止に有効 (sibling sync 契約)。
- **placeholder pair の value source 対称化**: `{related_page_title}` と `{related_page_path}` のような pair placeholder は、片方だけ詳細化すると drift を生む。両方の value source を同一 section で明示する規約が canonical。本 PR では 4.3 に「title 規約」サブセクションを追加して対称化。
