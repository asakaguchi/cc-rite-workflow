### Fix Patterns
- 未クォート `for x in $var` ループの noglob 化には `set -f`/`set +f` で囲うのが最小修正。`case`/`[[ == ]]` のパターンマッチは noglob 非影響なので検出ロジックを壊さない。直前の noglob 状態を `case $- in *f*)` で save/restore すると drift-safe。
- glob 汚染は「length-guard の後で展開されトークン数が bound されない」点が timeout→fail-open(RCE)の核。noglob 化するだけで over-DENY 誤検出と timeout 無制限反復の両方が同時に閉じる(反復キャップは不要になる)。
- コード allowlist と、それを deny メッセージが参照先指定する doc テーブルは cycle をまたいで語彙同期を保つ。非網羅リストには「等」を明示して閉じた列挙と誤読されないようにする。
- 兄弟の同型パターン(pre-existing)は revert test で本 PR 由来でないと判定し、propagate せず調査推奨に留める(scope discipline)。回帰テストは「revert で fail するか」を実証して有意性を担保する。
