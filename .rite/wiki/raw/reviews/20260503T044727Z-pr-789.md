## Key Patterns Encountered

### Asymmetric Fix Transcription (multi-cycle recurrence)

The dominant failure mode in this PR was **Asymmetric Fix Transcription propagation gaps** across cycles:
- Cycle 1 → 2: Inner sections updated, outer DRIFT-CHECK ANCHOR header (line 7) and 3 caller anchors missed
- Cycle 2 → 3: Caller anchor at create-interview.md:592 missed (inline anchor inside `Issue #634 enhancement` blockquote)
- Cycle 2 → 3: SoT-internal `4 occurrence` vs new `6 occurrence` table inconsistency

This validates the existing Wiki entry "Asymmetric Fix Transcription (対称位置への伝播漏れ)" — even sibling-grep verification failed to catch the inline anchor inside another blockquote (line 592) until cycle 3 review.

### Sub-site / Occurrence unit confusion

When introducing a new dimension (sub-site 3a/3b) to an existing count (4 site → 3 site + sub-structure), wording like "の N occurrence" in summary tables created false-positive consistency: the number `2 occurrence` matched at line 17 but counted sub-sites, not occurrences. The new occurrence-unit table (lines 47-55) made the discrepancy visible.

**Lesson**: When adding a unit (sub-site, occurrence, etc.), explicitly enumerate the cardinality in a separate column rather than embedding in prose.

### `--if-exists` arithmetic with mixed units

Mixing "site count" and "occurrence count" with arithmetic notation (`1+0+0 = 1 site`) created compound ambiguity. The fix was to split into two tables (occurrence-unit + site-unit) rather than reconcile in a single sentence.
