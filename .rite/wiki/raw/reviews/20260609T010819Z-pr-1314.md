- jq 検出 pipeline の behavioral test は「ソースから awk で実 program を抽出して fixture 実行」することで、静的 grep では検出不能な semantic break (jq literal 保持のまま scoping を反転) を捕捉できる。コピー fixture では本体と乖離するため、ソース抽出方式が優れる。

- awk 抽出は marker 一意性 + terminator 境界に構造依存するため、抽出失敗時の `[ -z "$jq_prog" ]` fail-safe ガードが silent pass を防ぐ。
