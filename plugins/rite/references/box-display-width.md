# 罫線 box の表示幅ルール（SoT）

コマンド出力で使う罫線 box（`┌─┐` / `│ … │` / `└─┘`）の**右罫線 `│` を揃える**ための
表示幅ルールの単一の Source of Truth。box を描画するコマンド（`commands/issue/list.md`,
`commands/workflow.md`, `commands/skill/suggest.md`, `commands/getting-started.md` 等）は
本ルールに従って内側のパディングを決める。

## 問題

右罫線のパディングを「**文字数（コードポイント数）**」で詰めると、全角文字を含む行で枠が崩れる。
`一覧` や全角括弧 `（）` などの East Asian Wide / Fullwidth 文字は 1 コードポイントで
**2 表示桁**を占めるため、行の実表示幅が文字数より広くなり、右罫線 `│` が左へずれる。
さらに filter 値が `open`（半角）と `バックログ`（全角）で行幅が変わるため、
固定パディングでは全パターンを揃えられない。

## ルール

1. **内側表示幅 = 上罫線の `─` 本数**。content 行 `│` + inner + `│` の `inner` の
   **表示幅**を、その box の上罫線（`┌─…─┐`）に含まれる `─` の本数に一致させる。
2. **表示幅の数え方**（1 文字あたりの桁数）:

   | East Asian Width | 区分 | 桁数 |
   |------------------|------|------|
   | `W` (Wide) / `F` (Fullwidth) | 全角（CJK・全角記号・全角括弧 `（）` 等） | **2** |
   | `Na` (Narrow) / `H` (Halfwidth) / `N` (Neutral) | 半角英数記号・半角カナ等 | 1 |
   | `A` (Ambiguous) | 曖昧幅（`→` `…` `○` 等） | **1** |

3. **パディングは右側のみ**で調整する。leading 空白（中央寄せ目的の左パディング）と
   コアテキストは保持し、`inner` の末尾空白だけを増減して内側表示幅を合わせる。

## Ambiguous 幅の注意（既知の制約）

`A`（Ambiguous）は端末・フォント設定により半角(1)/全角(2)のどちらにも描画され得る。
本ルールは**再現性を優先して一律 1 桁**として扱う（Python `unicodedata.east_asian_width`
の素の区分・modern wcwidth の既定と一致）。そのため CJK レガシー幅設定の端末では
`A` を含む box（例: `→` を含む Status 遷移図）が 1 桁ずつずれ得る。
**box の content には Ambiguous 文字をできるだけ使わない**ことで崩れを避けられる。
これは box 罫線方式の構造的なトレードオフであり、完全な端末非依存は保証しない。

## 検証

各 box の上罫線 `─` 本数を target とし、`│…│` 行の inner 表示幅（上表の桁数で算出）が
target と一致するかを照合する。フロー図など 2 列の ASCII アート（短い box / 複数 `│`）は
本ルールの対象外（単純な単列ヘッダー box のみが対象）。
