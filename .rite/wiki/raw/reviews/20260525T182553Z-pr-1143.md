### Key Pattern (Anti-pattern Wiki 候補 — Critical)

**Declarative invariant escalation trap**: SoT 集約 PR で「両者を集合等価に保つ」という declarative invariant note を追加すると、note の文言自体が新たな self-meta-conflict 経路を生む。具体的に:

1. invariant が **bidirectional set equality** を主張すると、片方向しか実際に成立していない場合の self-refuting 状態が発生
2. invariant の **section reference** が drift しやすい (cycle 4: § letter が wrong section を指す)
3. invariant 自身が **6 原則違反** に近い meta-violation を含む経路がある (line 20 cycle 2 修正、line 393 cycle 4 同型)

教訓: declarative invariant より **mechanical test (実行可能な parity verification script)** に置き換える方が、note の文言自体が新たな regression 源になる経路を断つ。Issue #1140 で 5 cycle 連続発生した本パターンは AI agent による反復的 fix の典型例。
