### Finding Patterns

Bash tool呼び出し間でシェル変数が保持されない、という rite workflow 全体で繰り返し観測される罠が今回も2件検出された:
1. あるステップで set したシェル変数を、別ステップ（別 Bash tool 呼び出し）が `[CONTEXT] KEY=value` sentinel なしに参照しようとする（projects_status_updated の例）。
2. `status_json=$(cmd) || status_json=""` という一見安全な fallback パターンが、cmd が非ゼロ終了時に stdout へ出力済みの有用な診断JSONを空文字列で上書き・破棄してしまう。この同一パターンが ready.md / issue-close.md / archive-procedures.md にも既存し、系統的な問題であることが判明した。
