# テスト仕様書: start.md Phase 5.1.1 完了後の必須アクション

## 概要

`plugins/rite/commands/issue/start.md` の Phase 5.1.1「変更のコミットとプッシュ」完了後に表示される必須アクションボックスに対するテスト仕様書。

## 対象機能

コミット・プッシュ完了後に AI が必ず `rite:lint` を呼び出すことを保証するための指示ボックス（`### 🚨 Phase 5.1.1 完了後の必須アクション（絶対厳守）`）の動作確認。

---

## テストケース

### TC-001: 必須アクションボックスの構造確認

**目的**: 必須アクションボックスが正しい構造と内容を持っていることを確認する。

**対象セクション** (`start.md` 行 2390-2403):

```markdown
### 🚨 Phase 5.1.1 完了後の必須アクション（絶対厳守）

> **警告**: 以下のアクションを実行せずに停止することは禁止されている。
>
> 1. コミット・プッシュが成功したことを確認
> 2. **即座に** Skill ツールで `rite:lint` を呼び出す
> 3. ユーザーに「次のステップ」を案内して停止してはならない

**フロー検証チェック（Phase 5.1.1 完了時に必ず確認）:**
- [ ] コミット完了
- [ ] プッシュ完了
- [ ] **次のアクション**: Skill ツールで `rite:lint` を呼び出す（**今すぐ実行**）
```

**検証項目**:

| # | 検証項目 | 期待値 | 結果 |
|---|---------|--------|------|
| 1 | 見出しレベル | `###`（H3）| - |
| 2 | 警告アイコン | 🚨 | - |
| 3 | 強調表現 | 「絶対厳守」「禁止」「即座に」「今すぐ実行」 | - |
| 4 | チェックリスト項目数 | 3 項目 | - |
| 5 | 次のアクション | `rite:lint` の呼び出し | - |

### TC-004: 誤動作パターンの防止確認

**目的**: 仕様書が AI の誤動作パターンを明示的に禁止していることを確認する。

**禁止される動作パターン**:

| # | 誤動作パターン | 仕様書での禁止表現 | 結果 |
|---|--------------|------------------|------|
| 1 | コミット後に停止 | 「停止することは禁止」 | - |
| 2 | ユーザーに「次は lint です」と案内して待機 | 「案内して停止してはならない」 | - |
| 3 | lint 呼び出し前に確認を求める | 「即座に」「今すぐ実行」 | - |
| 4 | 5.1.1 完了報告のみ出力 | 「以下のアクションを実行せずに停止することは禁止」 | - |

**手動検証手順**:

1. `/rite:issue:start` を実行
2. 実装・コミット・プッシュまで進める
3. AI の出力を観察:
   - ❌ NG パターン: 「コミット・プッシュが完了しました。次に lint を実行します。」で停止
   - ✅ OK パターン: コミット・プッシュ完了後、即座に `rite:lint` の Skill ツール呼び出しが行われる

## 改善提案

テストケース作成時に発見された仕様の課題:

### IP-001: フロー検証チェックリストの実行タイミング

**発見元**: TC-001, TC-005

**観察**: フロー検証チェックリストは AI が「確認」するものとして記載されているが、これは AI 自身のセルフチェック用であり、ユーザーへの表示ではない。

**現状**: 明示的に「AI 内部でのセルフチェック用」とは記載されていない。

**提案**: 以下の注釈を追加することを検討:

```markdown
**フロー検証チェック（AI 内部でのセルフチェック用）:**
```

**優先度**: Low（動作には影響しない）
