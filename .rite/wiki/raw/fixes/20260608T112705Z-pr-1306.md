### Fix Patterns
新規 lint check スクリプト (watchdog-status-mismatch.sh から作法踏襲) の cross-validated 2 件を 1 cycle で修正:
- F-01 (code-quality + error-handling cross-validated MEDIUM): peer script からの copy-paste 時に exit-code 契約の非対称を導入。watchdog は exit 1=fatal だが本 PR は exit 1=warning(drift)/exit 2=invocation-error と再設計。`--limit) LIMIT="${2:-}"; shift 2` の末尾値欠落で `set -e` 下 `shift 2` が exit 1 abort → usage error が lint 上 "drift 検出" warning に誤分類。fix: `[ "$#" -lt 2 ]` で値存在を gate し欠落時 exit 2 へ統一。教訓: peer script からの作法踏襲時は「copy-paste 同形性は exit-code 契約一致を保証しない」(PR #631 系譜)。`${2:-}` は set -u の unset 参照のみ防御し shift 2 の失敗は防げない。
- F-02 (test MEDIUM): T-6 の `out=$(...); rc=$?` が `set -euo pipefail` 下で被テストスクリプト回帰時に代入行で harness abort → 明示 failure 分岐が dead code 化 (failure attribution 破壊、silent pass ではない)。fix: `set +e; out=$(...); rc=$?; set -e` で囲む。mutation test (exit 2 を返す broken script) で FAIL 分岐到達を確認。canonical: command-substitution + `rc=$?` を `set -e` 下で書く時は明示的に set +e で囲む。
